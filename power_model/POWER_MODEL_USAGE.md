# Power Model Usage Guide

This guide explains how to calculate power and energy consumption for AstraSim simulations.

## Overview

The power model has four operational modes:
- **Mode A**: Baseline (no LPM)
- **Mode B**: Compute LPM only
- **Mode C**: Communication LPM only
- **Mode D**: Full LPM (energy-proportional)

## Quick Start

### 1. Prepare Input Files

You need three files (one is optional):

#### Compute Log (Required)
AstraSim generates a structured log with per-NPU timing:
```
[2026-03-01 23:11:40.576] [workload] [statistics] sys=0, Wall time: 1234567 GPU time: 1000000 Comm time: 200000
```

The parser extracts cycles (@ 1 GHz clock) for each NPU.

#### Network CSV (Required)
Link-traffic statistics with optional per-link bandwidth:
```csv
Source,Destination,Total_Bytes,Bandwidth (BG/s)
0,148,70258845696.00,1800.0
1,148,70258845696.00,1800.0
...
```

- **Source/Destination**: node IDs (prefixed by type: `h` for hosts, `t` for ToR, `a` for aggregation, `c` for core, `v` for NVSwitch, `n` for NIC)
- **Total_Bytes**: traffic volume on this link
- **Bandwidth (BG/s)**: link capacity in GB/s (optional; defaults to 450 GB/s if missing)

#### Nodemap JSON (Optional)
Topology definition for topology-aware switch power modeling:
```json
{
  "num_hosts": 64,
  "total_switches": 156,
  "gpus_per_node": 8,
  "hosts": ["h0", "h1", ...],
  "switches_by_type": {
    "nvswitch": ["v0", "v1", ...],
    "tor": ["t0", "t1", ...],
    "aggregation": ["a0", "a1", ...],
    "core": ["c0", "c1", ...]
  }
}
```

Generated by `nodemap_parser.py` or provided manually.

#### Power Config JSON (Optional)
Hardware parameters and LPM settings. See `a100_config.json` for a complete example.

Key sections:
- **GPU power**: `gpu_peak_power`, `gpu_idle_power`, `gpu_sleep_power`
- **Link power**: `nvlink_*`, `nic_*`, `fabric_*` (active/idle/sleep)
- **Switch power**: `switch_active_power`, `switch_idle_power`, `switch_sleep_power`
- **Switch degrees**: ports per switch for each type
  ```json
  "switch_degrees": {
    "nvswitch":    18,
    "tor":         64,
    "aggregation": 64,
    "core":        64
  }
  ```

### 2. Run the Power Calculator

```bash
python calculate_power.py \
  --compute /path/to/astrasim.log \
  --network /path/to/link_traffic.csv \
  --nodemap /path/to/nodemap.json \
  --config /path/to/power_config.json \
  --mode A
```

## Configuration Details

### Switch Degrees (Ports per Switch)

The power model supports two modes:

**1. User-specified (in config JSON):**
```json
"switch_degrees": {
  "nvswitch":    18,
  "tor":         64,
  "aggregation": 64,
  "core":        64
}
```

**2. Topology-derived (from NS3 file):**
If `switch_degrees` is empty or absent, the model averages the connectivity from the NS3 topology file:
```python
degree = average(neighbors per switch)
```

### Per-Link Bandwidth

The network parser supports three fallback strategies:

1. **CSV column** (highest priority): If `Bandwidth (BG/s)` column exists, parse and convert to Bytes/s
2. **Link object**: If CSV has no bandwidth, use the `LinkStats.bandwidth` field (from parser default)
3. **Model default**: If all else fails, use 450 GB/s

Example CSV with per-link bandwidth:
```csv
Source,Destination,Total_Bytes,Bandwidth (BG/s)
128,130,264355784704.00,400.0
148,0,70258845696.00,1800.0
```

### Power Calculation Formula

**Per-NPU GPU Energy:**
$$E_{GPU} = \sum_{i=0}^{N-1} P_i \times T_{wall,i}$$

Where:
- $P_i$ = power for NPU $i$ (depends on compute/comm utilization and LPM mode)
- $T_{wall,i}$ = wall-clock time for NPU $i$

**Per-Switch Dynamic Power:**
$$P_{dyn,s} = degree \times (P_{active} - P_{base}) \times \sum \min(u_{link}, 1.0)$$

Where:
- $degree$ = ports per switch (user-specified or topology-derived)
- $u_{link}$ = utilization of links touching switch $s$

## Python API

If you want to use the power model programmatically:

```python
from compute_parser import parse_astrasim_log
from network_parser import parse_network_statistics_csv
from power_config import PowerConfig
from power_model import PowerModel

# Parse inputs
compute = parse_astrasim_log('sim.log')
network = parse_network_statistics_csv(
    'link_traffic.csv',
    nodemap_file='nodemap.json',
    prefix_map={'nvswitch': 'v', 'tor': 't', ...}
)

# Create power model
config = PowerConfig.from_json('a100_config.json', mode='D')
model = PowerModel(compute, network, config)

# Get results
print(f"Total power: {model.total_power():.2f} W")
print(f"Total energy: {model.total_energy():.2f} J")
print(f"Samples/J: {model.samples_per_joule():.4f}")

breakdown = model.get_complete_breakdown()
print(breakdown)
```

## Troubleshooting

**"Error parsing link-traffic CSV"**
- Check CSV column names match exactly: `Source`, `Destination`, `Total_Bytes`
- Optional bandwidth column: `Bandwidth (BG/s)` or similar

**"Error parsing astrasim log"**
- Log must use spdlog structured format
- Check for lines like: `[statistics] sys[X], Wall time: N GPU time: N Comm time: N`

**"No topology_info; link classification will be 'unknown'"**
- Provide `--nodemap` file for topology-aware link type detection
- Without it, all fabric links default to generic fabric power params

**Switch degrees not as expected**
- Check `switch_degrees` in config JSON matches topology switch types
- If missing, model derives from NS3 topology file (if provided)
